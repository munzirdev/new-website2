# استكشاف أخطاء نموذج العودة الطوعية

## المشكلة الحالية
```
خطأ في حفظ النموذج
```

## خطوات التشخيص والحل

### 1. اختبار الاتصال بقاعدة البيانات

#### **استخدام زر "اختبار الاتصال":**
1. اذهب إلى صفحة نموذج العودة الطوعية
2. اضغط على زر **"اختبار الاتصال"** (أزرق)
3. راقب الرسالة التي تظهر

#### **النتائج المتوقعة:**
- ✅ **نجح:** "الاتصال بقاعدة البيانات يعمل بشكل صحيح"
- ❌ **فشل:** رسالة خطأ محددة

### 2. التحقق من تسجيل الدخول

#### **المتطلبات:**
- يجب أن تكون مسجل الدخول
- يجب أن يكون لديك حساب صالح

#### **كيفية التحقق:**
1. تأكد من أنك مسجل الدخول
2. تحقق من أن اسم المستخدم يظهر في الشريط العلوي
3. إذا لم تكن مسجل الدخول، سجل الدخول أولاً

### 3. التحقق من البيانات المطلوبة

#### **الحقول المطلوبة:**
- ✅ الاسم الكامل (تركي)
- ✅ الاسم الكامل (عربي)
- ✅ رقم الهوية
- ✅ نقطة العبور

#### **الحقول الاختيارية:**
- 📱 رقم الهاتف
- 📅 تاريخ مخصص
- 👥 المرافقون

### 4. مراقبة Console للأخطاء

#### **فتح Developer Tools:**
1. اضغط `F12` أو `Ctrl+Shift+I`
2. اذهب إلى تبويب **Console**
3. حاول حفظ النموذج
4. راقب الرسائل التي تظهر

#### **الرسائل المتوقعة:**
```
🔍 بدء عملية حفظ النموذج...
📊 بيانات النموذج: {...}
👥 المرافقون الصالحون: [...]
💾 بيانات الحفظ: {...}
✅ تم الحفظ بنجاح: {...}
```

### 5. الأخطاء الشائعة وحلولها

#### **❌ خطأ في المصادقة:**
```
خطأ في المصادقة - يرجى إعادة تسجيل الدخول
```
**الحل:**
- أعد تسجيل الدخول
- تأكد من صحة بيانات الحساب

#### **❌ بيانات النموذج غير مكتملة:**
```
جميع الحقول المطلوبة يجب أن تكون مملوءة
```
**الحل:**
- املأ جميع الحقول المطلوبة
- تأكد من عدم ترك أي حقل فارغ

#### **❌ خطأ في الاتصال بقاعدة البيانات:**
```
خطأ في الاتصال بقاعدة البيانات: ...
```
**الحل:**
- تحقق من اتصال الإنترنت
- تأكد من أن خادم Supabase يعمل
- تحقق من متغيرات البيئة

#### **❌ خطأ غير متوقع:**
```
خطأ غير متوقع في حفظ النموذج
```
**الحل:**
- أعد تحميل الصفحة
- حاول مرة أخرى
- تحقق من Console للأخطاء التفصيلية

### 6. التحقق من قاعدة البيانات

#### **جدول voluntary_return_forms:**
```sql
-- التحقق من وجود الجدول
SELECT EXISTS (
    SELECT FROM information_schema.tables 
    WHERE table_name = 'voluntary_return_forms'
);

-- التحقق من الصلاحيات
SELECT grantee, privilege_type 
FROM information_schema.role_table_grants 
WHERE table_name = 'voluntary_return_forms';
```

#### **السياسات (Policies):**
```sql
-- التحقق من سياسات RLS
SELECT * FROM pg_policies 
WHERE tablename = 'voluntary_return_forms';
```

### 7. اختبار الحفظ خطوة بخطوة

#### **الخطوة 1: اختبار الاتصال**
1. اضغط "اختبار الاتصال"
2. تأكد من نجاح الاتصال

#### **الخطوة 2: ملء البيانات**
1. املأ جميع الحقول المطلوبة
2. تأكد من صحة البيانات

#### **الخطوة 3: حفظ النموذج**
1. اضغط "حفظ النموذج"
2. راقب الرسائل في Console
3. تحقق من الرسالة التي تظهر

### 8. معلومات التصحيح

#### **بيانات التصحيح المطلوبة:**
- رسالة الخطأ الكاملة
- لقطة شاشة من Console
- بيانات النموذج التي تحاول حفظها
- حالة تسجيل الدخول

#### **كيفية جمع المعلومات:**
1. افتح Developer Tools (F12)
2. اذهب إلى Console
3. حاول حفظ النموذج
4. انسخ جميع الرسائل التي تظهر
5. التقط لقطة شاشة من النموذج

### 9. الحلول البديلة

#### **إذا استمرت المشكلة:**
1. **أعد تشغيل الخادم:**
   ```bash
   npm run dev
   ```

2. **تحقق من ملف .env:**
   ```env
   VITE_SUPABASE_URL=...
   VITE_SUPABASE_ANON_KEY=...
   ```

3. **أعد تسجيل الدخول:**
   - سجل الخروج
   - أعد تسجيل الدخول

4. **تحقق من متصفح آخر:**
   - جرب متصفح مختلف
   - امسح Cache و Cookies

### 10. الاتصال بالدعم

#### **إذا لم تحل المشكلة:**
1. جمع جميع معلومات التصحيح
2. وصف الخطوات التي اتبعتها
3. إرفاق رسائل الخطأ
4. تواصل مع فريق الدعم

## ملاحظات مهمة

- ✅ **اختبار الاتصال** يساعد في تحديد المشكلة
- ✅ **Console** يحتوي على معلومات تفصيلية
- ✅ **جميع الحقول المطلوبة** يجب أن تكون مملوءة
- ✅ **تسجيل الدخول** مطلوب للحفظ
- ✅ **الاتصال بالإنترنت** ضروري للعمل
