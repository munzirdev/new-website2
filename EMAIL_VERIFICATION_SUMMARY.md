## 🚀 كيفية الإعداد

### الخطوة 1: إعداد Supabase Dashboard
1. اذهب إلى [Supabase Dashboard](https://supabase.com/dashboard)
2. اختر مشروعك
3. اذهب إلى **Settings** > **Auth** > **General**
4. فعّل **Enable email confirmations**
5. عيّن **Site URL** إلى: `https://tevasul.group`

### الخطوة 2: إعداد SMTP
1. اذهب إلى **Settings** > **Auth** > **Email Templates**
2. فعّل **Custom SMTP**
3. أدخل معلومات SMTP:
   ```
   SMTP Host: smtp.gmail.com
   SMTP Port: 587
   SMTP User: your-email@gmail.com
   SMTP Pass: your-app-password
   Encryption: STARTTLS
   ```

### الخطوة 3: تشغيل ملف SQL
1. في **SQL Editor**، انسخ محتوى `enable-email-verification.sql`
2. شغّل الكود
3. استبدل `your-email@gmail.com` و `your-app-password` بالقيم الصحيحة

### الخطوة 4: نشر Edge Functions
```bash
supabase functions deploy send-verification-email
supabase functions deploy resend-verification
```

### الخطوة 5: اختبار النظام
1. أنشئ حساب جديد
2. تحقق من وصول بريد التحقق
3. اضغط على رابط التأكيد
4. اختبر إعادة إرسال البريد الإلكتروني

## 🔧 الميزات المضافة

- رسائل تأكيد واضحة

### 4. **تتبع وإحصائيات**

## 🔍 استكشاف الأخطاء

### المشاكل الشائعة:

**❌ البريد الإلكتروني لا يصل**
- تحقق من إعدادات SMTP
- تأكد من صحة كلمة مرور التطبيق
- راجع سجلات Supabase

**❌ البريد الإلكتروني في Spam**
- أضف سجلات SPF وDKIM
- تحقق من إعدادات المرسل

**❌ خطأ في الاتصال**
- تحقق من المنفذ (587)
- تأكد من تفعيل STARTTLS

## 📱 تجربة المستخدم

### 1. **إنشاء الحساب**
1. المستخدم يملأ نموذج التسجيل
2. يتم إرسال بريد التحقق تلقائياً
3. تظهر رسالة تأكيد للمستخدم

### 2. **تأكيد البريد الإلكتروني**
1. المستخدم يفتح بريده الإلكتروني
2. يضغط على رابط التأكيد
3. يتم توجيهه إلى صفحة التأكيد
4. يمكنه إعادة إرسال البريد إذا لم يصل

### 3. **تسجيل الدخول**
1. بعد التأكيد، يمكن للمستخدم تسجيل الدخول
2. يتم توجيهه إلى الصفحة الرئيسية
3. تظهر رسالة ترحيب
